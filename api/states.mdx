Retrieve land suitability data for an entire US state across all four analysis modes. This is useful for quickly screening which states are worth analyzing at the property level before spending report quota. The endpoint is free — it does not consume any of your monthly quota.

<Note>
  This endpoint requires authentication but does not consume report quota. Use it freely to screen states before submitting individual property analyses.
</Note>

## Authentication

Include your API key as a Bearer token in the `Authorization` header.

```bash
Authorization: Bearer ls_live_YOUR_API_KEY
```

## Request examples

<CodeGroup>

```bash All modes
curl https://api.landscout.io/v1/states/NM \
  -H "Authorization: Bearer ls_live_YOUR_API_KEY"
```

```bash Single mode
curl "https://api.landscout.io/v1/states/NM?mode=investment" \
  -H "Authorization: Bearer ls_live_YOUR_API_KEY"
```

</CodeGroup>

## Path parameters

<ParamField path="code" type="string" required>
  Two-letter US state code (e.g., `"NM"`, `"TX"`, `"CO"`). Case-insensitive. Returns `404` for unrecognized codes.
</ParamField>

## Query parameters

<ParamField query="mode" type="string">
  Filter the response to a single analysis mode. Accepted values: `off_grid`, `rural_residential`, `recreational`, `investment`. Omit this parameter to return all four modes.
</ParamField>

## Response fields

<ResponseField name="state_code" type="string" required>
  The two-letter US state code in uppercase (e.g., `"NM"`).
</ResponseField>

<ResponseField name="modes" type="object" required>
  Contains one key per mode returned. If you omit the `mode` query parameter, all four keys are present. If you filter to a single mode, only that key is present.

  <Expandable title="mode object properties">
    <ResponseField name="overall_score" type="number" required>
      Overall suitability score for this mode, from 0 to 100. Higher is better.
    </ResponseField>

    <ResponseField name="sub_scores" type="object" required>
      Per-category scores (0–100) specific to each mode. Keys vary:

      - **off_grid**: `solar`, `water`, `septic`, `building_codes`, `access`
      - **rural_residential**: `utilities`, `build_cost`, `flood_fire_risk`, `permit_complexity`, `access`
      - **recreational**: `terrain`, `water_features`, `cover`, `neighboring_use`, `seasonal_access`
      - **investment**: `zoning`, `comps`, `price_per_acre`, `holding_cost`, `development_potential`
    </ResponseField>

    <ResponseField name="summary" type="string" required>
      A narrative summary of the state's suitability for this mode.
    </ResponseField>

    <ResponseField name="key_regulations" type="string[]" required>
      A list of notable state-level regulations or policies relevant to this mode (e.g., water rights rules, zoning preemption laws).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="shared_facts" type="object" required>
  State-wide data points that apply across all modes, such as tax rates.

  <Expandable title="shared_facts properties">
    <ResponseField name="state_income_tax" type="string">
      State income tax rate or structure (e.g., `"4.9%"`).
    </ResponseField>

    <ResponseField name="property_tax_avg" type="string">
      Average effective property tax rate for the state (e.g., `"0.78%"`).
    </ResponseField>
  </Expandable>
</ResponseField>

## Errors

| HTTP status | Error code | Cause |
|-------------|------------|-------|
| `401` | `unauthorized` | Missing, malformed, or revoked API key. |
| `403` | `forbidden` | Your account is suspended or has a past-due invoice. |
| `404` | `not_found` | The state code is not a valid two-letter US state abbreviation. |

See the [error reference](/reference/errors) for the full error response shape.
