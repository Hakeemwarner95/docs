Check how many reports you have used in the current billing period, how many remain, and whether you have incurred any overage charges. This endpoint is free — it does not consume any of your monthly quota.

<Note>
  The same quota information is also returned on every API response via the `X-Quota-Limit` and `X-Quota-Remaining` headers. Check those headers to avoid an extra round-trip in latency-sensitive code paths.
</Note>

## Authentication

Include your API key as a Bearer token in the `Authorization` header.

```bash
Authorization: Bearer ls_live_YOUR_API_KEY
```

## Request example

```bash
curl https://api.landscout.io/v1/usage \
  -H "Authorization: Bearer ls_live_YOUR_API_KEY"
```

## Response fields

<ResponseField name="period_start" type="string" required>
  ISO 8601 UTC timestamp for the start of your current billing period (e.g., `"2026-04-01T00:00:00Z"`).
</ResponseField>

<ResponseField name="period_end" type="string" required>
  ISO 8601 UTC timestamp for the end of your current billing period (e.g., `"2026-04-30T23:59:59Z"`).
</ResponseField>

<ResponseField name="quota_limit" type="number" required>
  Total reports included in your plan for this billing period.
</ResponseField>

<ResponseField name="quota_used" type="number" required>
  Reports consumed so far this billing period.
</ResponseField>

<ResponseField name="quota_remaining" type="number" required>
  Reports remaining before you hit your plan limit (`quota_limit - quota_used`). Does not account for overage — see `overage_units` for usage above quota.
</ResponseField>

<ResponseField name="overage_units" type="number" required>
  Reports consumed above your plan quota this period. Always `0` on the Developer tier, which hard-blocks at quota. On Starter and above, overage reports are billed at metered rates via Stripe at the end of the billing period.
</ResponseField>

<ResponseField name="tier" type="string" required>
  Your current plan tier. One of: `developer`, `starter`, `growth`, `scale`, `enterprise`.
</ResponseField>

## Example response

```json
{
  "period_start": "2026-04-01T00:00:00Z",
  "period_end": "2026-04-30T23:59:59Z",
  "quota_limit": 100,
  "quota_used": 73,
  "quota_remaining": 27,
  "overage_units": 0,
  "tier": "starter"
}
```

<Tip>
  Every LandScout API response includes `X-Quota-Limit` and `X-Quota-Remaining` headers. Reading these headers on your existing requests is more efficient than calling `GET /v1/usage` separately — the values are identical.
</Tip>

## Errors

| HTTP status | Error code | Cause |
|-------------|------------|-------|
| `401` | `unauthorized` | Missing, malformed, or revoked API key. |
| `403` | `forbidden` | Your account is suspended or has a past-due invoice. |

See the [error reference](/reference/errors) for the full error response shape.
